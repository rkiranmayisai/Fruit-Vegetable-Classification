1. Digital Preprocessing Pipeline (Crucial Step)

To match the mathematical distribution of the MNIST dataset, drawings drawn at 450x450px undergo these transformations:

  • **Connected Component Analysis (CCA)**: Extracts bounding boxes around clusters of pixels, isolating distinct strokes.
  • **Normalizing Size**: The isolated digit box is scaled to fit exactly within a 20x20 pixel bounding grid, preserving its original aspect ratio.
  • **Centering**: The 20x20 box is embedded within a blank 28x28 grayscale canvas. Using the *center of mass* (centroid) of the active pixels, the model shifts the digit's coordinates so the centroid aligns perfectly at pixel `(14, 14)`. This centering eliminates translation variance.

2. Machine Learning Architecture

The core classifier uses a dense multinomial classifier layer trained on MNIST. Through the **Model Hub** tab, this model can be fine-tuned locally using browser IndexedDB storage and an adaptive learning loop.

3. Backpropagation-Based XAI (Saliency Map)

Explainability uses *gradient saliency*. By computing the Jacobian matrix of gradients of the predicted class score with respect to input pixels: $$\text{Saliency} = \left| \frac{\partial y_{\text{class}}}{\partial X_{\text{input}}} \right|$$ The interface highlights the exact strokes and pixels that influenced the weights, letting users inspect what features (e.g. loops, endpoints) the model is looking at.

Implementation FAQ

Why does the model sometimes misclassify digits?

Standard MNIST digits are centered and anti-aliased. If you write too quickly, draw multi-stroke detached digits, or make very thin strokes, the density distribution deviates from the training dataset. Try enabling the **Auto Center** setting or use thicker brushes.

How does the Mathematical Solver work?

In Equation Mode, our segmentation algorithm extracts individual components and sorts them left-to-right. It evaluates size/aspect-ratio metrics: items that are extremely wide and thin are classified as minus `-`, items with intersection points are classified as plus `+`, etc. The remaining are predicted as digits, and JavaScript evaluates the equation.

Where is my history saved?

All user authentication profiles, settings, and prediction history are stored inside your browser's local sandbox via `localStorage`. Clearing the browser history or clicking 'Reset Database' clears this information.
